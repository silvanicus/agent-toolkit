# Competency Framework

This file defines expected behaviors and skills by role and level. Skills and agents reference this file to anchor interview guides, feedback, and calibration.

Levels: **Junior** · **Mid** · **Senior** · **Staff/Lead**
Dimensions: **Technical** · **Delivery** · **Collaboration** · **Growth**

---

## Frontend Engineer

### Junior
- **Technical**: Solid HTML/CSS/JS fundamentals. Builds UI from design specs using a component framework (React or equivalent). Writes basic unit tests.
- **Collaboration**: Participates in standups and reviews. Asks questions before getting stuck. Receives feedback constructively.

### Mid
- **Technical**: Proficient in React and TypeScript. Implements accessible, responsive UIs independently. Understands performance trade-offs (bundle size, lazy loading). Writes component and integration tests.
- **Collaboration**: Gives useful code review feedback. Works effectively with designers and backend engineers. Communicates status proactively.

### Senior
- **Technical**: Deep expertise in component architecture, state management, and performance. Sets coding standards and patterns. Leads accessibility and design system work. Reviews and validates AI-generated code with strong judgment.
- **Collaboration**: Drives technical decisions in cross-functional teams. Mentors junior and mid engineers. Communicates trade-offs to non-technical stakeholders. Comfortable in client-facing conversations.

### Staff
- **Technical**: Shapes the technical direction of the frontend discipline. Defines standards across projects. Evaluates major architectural changes and tooling decisions.
- **Collaboration**: Influences without formal authority. Sponsors junior and mid engineers. Builds alignment across engineering, product, and design.

## QA Engineer

### Junior
- **Technical**: Understands testing fundamentals (unit, integration, E2E). Writes manual test cases from requirements. Familiar with at least one automation framework. Reports bugs with clear reproduction steps.
- **Collaboration**: Works with developers to reproduce and verify bugs. Communicates blockers early.

### Mid
- **Technical**: Writes automated test suites (Playwright, Cypress, or equivalent). Integrates tests into CI/CD pipelines. Designs test plans for medium-complexity features. Can do basic API testing.
- **Collaboration**: Flags testability and acceptance criteria gaps during refinement. Reviews test cases with developers. Works with PMs on testing scope.

### Senior
- **Technical**: Designs and owns test automation strategy for a project. Expertise in performance, accessibility, and basic security testing. Leads test coverage reviews. Evaluates testing tools.
- **Collaboration**: Embeds quality thinking into the team's development process. Mentors junior QA engineers. Collaborates with clients on acceptance criteria and UAT.

### Lead
- **Technical**: Sets testing and quality strategy across a team or portfolio. Deep understanding of testing at scale. Decisions on tooling and standards.
- **Collaboration**: Builds QA capability in engineering teams. Acts as quality advocate in planning and estimation. Partners with client stakeholders on quality expectations.
